# reinforcement-14-employment-contractor-core-dev

## 用户问题

"我们找了一个外包工程师长期远程开发核心模块，每周参加例会，按月结算，也有代码仓库权限。现在他准备离开，还说核心代码有一部分是他的。我们要怎么处理？"

---

## 1. 一句话结论

这个"外包工程师"的工作模式（长期远程、参加例会、按月结算、接触核心代码）存在事实劳动关系和知识产权归属的双重风险——离开前必须先回收权限、固定证据、明确代码归属，否则核心资产可能失控。

## 2. 涉及领域

- **主领域**：employment（用工关系性质判断——外包还是事实劳动关系）、ip（核心代码归属、职务作品 vs 委托作品）
- **联动领域**：contracts（外包协议/劳动合同的权属条款）、disputes（离职争议、代码归属争议的证据固定）
- **核验领域**：governance（如果是联合创始人级别的核心人员，还涉及股权和治理问题）

## 3. 关键事实缺口 / 当前假设

| 缺口 | 当前假设 | 影响说明 |
|------|----------|----------|
| 签的是什么合同 | 假设签了外包/劳务协议，但权属条款不清楚 | 如果没有明确约定代码归属，法律上可能按委托作品处理，著作权不一定归你们 |
| 他主张的是哪部分代码 | 假设他参与了核心模块的架构设计和关键实现 | 如果是他独立编写且合同未约定归属，他的主张可能有法律依据 |
| 代码仓库是否有完整提交记录 | 假设 Git 提交记录可以追溯 | 提交记录是判断谁写了什么代码的关键证据 |
| 是否有保密和竞业协议 | 假设没有或保密条款不完善 | 没有保密约束，他可以把代码逻辑和业务理解带到下一个项目 |
| 实际管理方式 | 长期远程+每周例会+按月结算 | 这种管理模式更接近事实劳动关系，而不是独立的外包 |
| 是否缴纳社保 | 假设未缴纳 | 如果被认定为劳动关系，未缴社保可能引发劳动仲裁 |

## 4. 红线风险与风险等级

| 风险 | 等级 | 说明 |
|------|------|------|
| 核心代码权属争议 | 🔴 高 | 如果合同未明确约定代码归属，他可能依法享有著作权；核心模块代码如果被他带走或主张权利，对产品是致命打击 |
| 事实劳动关系风险 | 🔴 高 | 长期固定合作+参加例会+按月结算，可能被认定为事实劳动关系，他可以主张双倍工资、经济补偿和社保补缴 |
| 代码仓库权限未回收 | 🔴 高 | 他仍有代码仓库访问权限，可能在离开前复制、删除或修改核心代码 |
| 没有保密和竞业约束 | 🔴 高 | 他掌握核心代码逻辑和架构，如果没有保密协议，可以把这些知识带到竞品公司 |
| 离职交接不清 | 🟡 中 | 如果没有系统化的交接清单和确认，后续他可能主张还有未移交的代码或资料 |

## 5. 现在就做什么（24 小时内）

1. **立即回收所有代码仓库权限**：
   - 暂停或修改他的 Git 访问权限
   - 回收服务器、数据库、API Key、VPN、云平台等所有访问权限
   - 更换相关密码和密钥

2. **备份代码仓库当前状态**：
   - 立即对代码仓库做完整备份（包括所有分支和提交历史）
   - 导出 Git 日志，确认每一行代码的提交者、时间和修改内容
   - 保存备份的哈希值和时间戳

3. **不要在沟通中承认他对代码的所有权**：
   - 在明确法律判断之前，不要说"你的代码"或承认他拥有代码权利
   - 但也不要激化矛盾，保持专业和友好的沟通态度

4. **找到并仔细阅读你们签的合同**：
   - 确认合同标题是什么（外包/劳务/技术开发）
   - 确认有没有知识产权归属条款
   - 确认有没有保密条款
   - 如果找不到合同，这个风险等级要上调

## 6. 接下来怎么做（7 天 / 30 天）

### 7 天内

1. **完成正式的离职交接**：
   - 列出交接清单：所有代码、文档、架构设计、技术方案、账号密码、访问权限
   - 要求他签署交接确认单，明确所有工作成果已移交
   - 保留完整的交接记录

2. **签署保密和权利确认协议**：
   - **保密协议**：明确他接触过的所有代码、架构、业务逻辑、客户信息属于公司商业秘密，不得泄露或用于其他项目
   - **权利确认**：明确他在合作期间产出的所有代码、文档、技术方案的著作权和所有权归公司所有（即使原合同没有写，也通过这个协议补充确认）
   - 如果他不同意签署，这是严重的风险信号，需要立即咨询律师

3. **评估用工关系风险**：
   - 如果他的工作模式更接近劳动关系（长期、固定、受管理、月薪），需要评估他提起劳动仲裁的可能性
   - 如果存在较高的劳动关系认定风险，应提前准备应对方案，包括补缴社保的预算

### 30 天内

1. **完善外包/协作合同模板**：
   - 明确约定"合作期间产出的所有代码、文档、技术方案、设计稿等知识产权全部归公司所有"
   - 加入保密条款和竞业限制（对核心外包人员）
   - 明确交付标准和交接要求
   - 区分背景 IP 和新产出

2. **建立代码权限管理规范**：
   - 核心代码仓库权限分级管理
   - 外部协作者使用最小权限原则
   - 定期审查权限分配

3. **排查其他外包/兼职人员的合同和权限**：
   - 确认所有外部协作者都有签署权属和保密条款
   - 确认离职人员的权限已全部回收

## 7. 何时必须升级给外部律师 / 会计师 / 专业机构

- 他拒绝签署保密和权利确认协议
- 他正式主张代码著作权或要求支付额外费用
- 他提起劳动仲裁，主张事实劳动关系
- 发现他已将核心代码或技术方案带到了其他公司
- 他表示要自行使用或开源核心模块代码
- 涉及的代码价值较高，对产品有重大影响

## 8. 法律依据与适用层级

| 依据 | 层级 | 适用说明 |
|------|------|----------|
| 《著作权法》第11条 | 全国法律 | 著作权属于作者；法人作品除外 |
| 《著作权法》第18条 | 全国法律 | 职务作品：为完成法人工作任务所创作的作品，著作权由作者享有，但法人有权在其业务范围内优先使用；另有约定的除外 |
| 《著作权法》第19条 | 全国法律 | 委托作品：委托创作作品的著作权归属由委托人和受托人约定；未约定的，著作权属于受托人 |
| 《劳动合同法》第7条 | 全国法律 | 用人单位自用工之日起即与劳动者建立劳动关系 |
| 《劳动合同法》第10条 | 全国法律 | 建立劳动关系应当订立书面劳动合同；超过一个月未签的，应支付双倍工资 |
| 《劳动合同法》第82条 | 全国法律 | 超过一个月不满一年未签劳动合同的双倍工资罚则 |
| 劳社部发〔2005〕12号 | 部门规范性文件 | 确认劳动关系的三要素：主体合法、管理从属性、业务组成部分 |
| 《反不正当竞争法》第9条 | 全国法律 | 商业秘密保护；核心代码和架构如果符合秘密性、价值性和保密措施，可以作为商业秘密保护 |

**注意**：关键法律判断是——你们之间是委托关系还是劳动关系。如果是委托关系，合同没约定代码归属的，著作权默认归他（受托人）；如果是劳动关系，代码属于职务作品，控制力更强。他的实际工作模式（长期、例会、按月结算）增加了被认定为劳动关系的可能性。建议在采取正式法律行动前由执业律师评估。
