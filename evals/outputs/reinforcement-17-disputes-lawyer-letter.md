# reinforcement-17-disputes-lawyer-letter

## 用户问题

"我们收到一封律师函，说我们的 SaaS 抄袭了对方功能设计和页面，还要求 3 天内停止服务并赔 30 万。我们现在应该先回复还是先下架？"

---

## 1. 一句话结论

收到律师函不等于被起诉——先不要急着下架或赔钱，先做四件事：核实对方的权利基础、比对被指抄袭的内容、评估自己的抗辩空间，再决定是回复、谈判还是准备应诉，3 天期限通常不是法定期限，不必被对方的紧迫感带节奏。

## 2. 涉及领域

- **主领域**：disputes（律师函应对、证据固定、争议策略）
- **联动领域**：ip（功能设计和页面的知识产权保护范围、侵权判断）、contracts（客户合同中的服务连续性保障）
- **核验领域**：regulatory（如果对方同时向监管部门投诉）

## 3. 关键事实缺口 / 当前假设

| 缺口 | 当前假设 | 影响说明 |
|------|----------|----------|
| 对方的权利基础 | 假设对方声称有著作权/专利，但具体权利凭证待核验 | 对方是否有有效权利是判断侵权的前提 |
| "功能设计和页面"具体指什么 | 假设对方指出了某些功能流程和 UI 设计相似 | 需要逐项对比——功能设计（思想）通常不受保护，但具体的界面表达可能受保护 |
| 我们是否有独立开发的证据 | 假设有开发过程记录但不够系统 | 独立开发是最有力的抗辩，但需要证据支撑 |
| 律师函的发送主体 | 假设是对方委托的律师事务所 | 需要核验律所和律师的真实性 |
| 是否有接触过对方产品的可能性 | 假设双方在同一赛道 | 如果对方产品已公开，"接触"容易被推定 |
| 我们的客户是否会受影响 | 假设 SaaS 有在用客户 | 如果下架服务，客户权益受损可能引发违约 |

## 4. 红线风险与风险等级

| 风险 | 等级 | 说明 |
|------|------|------|
| 对方真的起诉并申请行为保全 | 🔴 高 | 法院可能在诉前或诉中裁定暂停服务，影响全部客户 |
| 在未核实前就下架或回复承认 | 🔴 高 | 主动下架可能被视为默认侵权；回复中不当表述可能构成自认 |
| 30 万赔偿是否在合理范围 | 🔴 高 | 需要评估对方的实际损失或你们的获利情况 |
| 对方同时向客户或平台投诉 | 🟡 中 | 可能导致客户恐慌或平台介入 |
| 销毁开发过程证据 | 🟡 中 | 如果在压力下删除相关文档，后续无法证明独立开发 |

## 5. 现在就做什么（24 小时内）

**不要急着下架，也不要急着回复。** 先做以下四件事：

### 第一步：核实律师函本身

1. **核实律所和律师的真实性**：在司法局官网查询律所和律师执业信息
2. **核实对方主体身份**：确认发函方的公司名称、统一社会信用代码是否真实
3. **确认函件是否为正式律师函**：是否有律所盖章、律师签名

### 第二步：固定证据

1. **立即保存你们产品的当前状态**：
   - 全页面截图或录屏（包括被指抄袭的功能和页面）
   - 保存产品版本号、发布时间和更新记录
2. **整理独立开发证据**：
   - Git 提交记录、设计稿版本历史、产品需求文档、会议纪要
   - 开发者证言（谁在什么时候参与了哪些功能的设计和开发）
3. **保存律师函原件**：原件扫描存档，不要在原件上做标记

### 第三步：初步对比

1. **逐项对比对方指出的"抄袭"内容**：
   - 功能流程：功能本身（思想层面）不受著作权保护，但具体的操作流程设计可能涉及
   - UI 界面：整体视觉效果、布局、配色、图标、交互动画等表达层面是否实质性相似
   - 代码：如果对方主张代码抄袭，需要逐行比对
2. **区分"行业通用设计"和"独创表达"**：
   - SaaS 行业的某些标准功能流程（如登录、列表、表单）属于通用设计，不受保护
   - 但具体的 UI 设计如果高度一致，可能构成侵权

### 第四步：评估选项

| 情况 | 建议 |
|------|------|
| 确实高度相似，抗辩空间小 | 考虑主动替换争议设计，启动谈判降低赔偿 |
| 部分相似但属于行业通用设计 | 准备抗辩理由，必要时请律师回复 |
| 完全是独立开发，不构成侵权 | 请律师正式回函说明独立开发事实 |

## 6. 接下来怎么做（7 天 / 30 天）

### 7 天内

1. **决定是否回复律师函**：
   - 如果决定回复，由律师起草正式回函，内容应包括：
     - 已收到函件
     - 对对方主张的初步回应（不承认侵权，说明独立开发）
     - 要求对方提供具体权利凭证和侵权对比细节
   - 如果不确定是否回复，可以不回复——律师函不是法律文书，不回复不产生法律后果

2. **准备替代方案**：
   - 如果最终需要调整，提前准备好 UI 和功能设计的替代方案
   - 确保替代方案不涉及对方的设计元素
   - 评估替换的成本和对客户的影响

3. **评估客户风险**：
   - 检查客户合同中是否有服务连续性承诺
   - 如果被迫暂停服务，准备客户沟通方案
   - 提前规划服务迁移或应急方案

### 30 天内

1. **根据对方后续动作决定策略**：
   - 如果对方没有进一步行动：继续使用产品，但完成 UI 差异化改造
   - 如果对方发送起诉状：立即聘请律师应诉，同时申请延长答辩期
   - 如果对方愿意协商：在律师协助下谈判和解方案

2. **完善产品设计的知识产权保护**：
   - UI 设计做著作权登记
   - 核心功能流程评估是否可申请专利
   - 建立设计留痕机制（设计稿版本、评审记录、修改历史）

3. **排查全产品线**：
   - 其他功能和页面是否也存在类似风险
   - 竞品对比，提前识别潜在争议点

## 7. 何时必须升级给外部律师 / 会计师 / 专业机构

- 收到法院起诉状或仲裁通知
- 对方申请了诉前行为保全（可能导致服务被强制暂停）
- 对方同时向你的客户、平台或监管部门发起投诉
- 30 万赔偿金额超出你们能自行协商解决的范围
- 对比后发现确实存在较高的侵权风险
- 对方律师函由知名知识产权律所发出，说明对方准备充分

## 8. 法律依据与适用层级

| 依据 | 层级 | 适用说明 |
|------|------|----------|
| 《著作权法》第3条 | 全国法律 | 计算机软件和美术作品受著作权保护；UI 界面可能作为美术作品保护 |
| 《著作权法》第10条 | 全国法律 | 复制权、信息网络传播权等 |
| 《著作权法》第24条 | 全国法律 | 合理使用的具体情形 |
| 《著作权法》第52-54条 | 全国法律 | 侵权责任和赔偿计算方式 |
| 《反不正当竞争法》第6条 | 全国法律 | 擅自使用他人有一定影响的包装、装潢等相同或近似的标识，引人误认的，构成不正当竞争 |
| 《反不正当竞争法》第12条 | 全国法律 | 利用技术手段实施的妨碍、破坏行为 |
| 《最高人民法院关于审理著作权民事纠纷案件适用法律若干问题的解释》 | 司法解释 | "接触+实质性相似"的侵权判断方法 |
| 思想与表达二分法 | 法理原则 | 著作权保护表达，不保护功能、思想和方法 |

**注意**：SaaS 产品的"功能设计"通常属于思想层面，不受著作权保护；但具体的 UI 界面表达如果构成实质性相似，可能构成侵权。此外，如果对方的页面设计有一定市场影响力，还可能触发《反不正当竞争法》第6条的混淆条款。建议在回复或应诉前由知识产权律师做详细的法律评估。
