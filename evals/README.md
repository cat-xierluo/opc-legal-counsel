# 评测样本说明

本目录用于存放 `opc-legal-counsel` 的回归评测样本。

当前采用两层分组：

- `foundation`：验证技能不会因为强调 OPC / AI 而丢失小微企业基础盘
- `reinforcement`：验证技能能在 OPC / AI 场景下给出更强的专项判断

当前 24 条样本重点检查以下 7 个维度：

1. 是否命中合理的多领域路由
2. 是否先补关键事实缺口，而不是直接硬答
3. 是否保持固定输出协议的核心结构
4. 是否正确区分全国基线与地方覆盖层
5. 是否在高风险事件中明确触发升级边界
6. 是否命中成长阶段模块或行业 overlay 的正确“下一跳”
7. 是否在高频场景中调用合适的样稿或输出资产

后续扩容时，应额外检查：

8. 是否体现跨领域综合判断，而不是按单一领域割裂回答

`evals/evals.json` 目前先记录：

- 用户问题
- 所属层级
- 预期路由
- 预期下一跳
- 地方覆盖层预期
- 升级边界预期
- 推荐样稿或输出资产
- 回答要点

`evals/assertions.json` 目前记录：

- 10 个重点样本的机器可读断言
- 每个样本最低通过断言数
- 关键词命中、分组关键词命中和禁用表述检查
- 回答文件约定：`<outputs-dir>/<eval_id>.md`

人工复核说明：

- `manual-review.md`：10 个重点样本的 10 分制评分说明、硬失败条件和复核记录格式

脚本检查：

- 运行结构检查：`python3 scripts/check-evals.py`
- 检查实际回答：`python3 scripts/check-evals.py --outputs-dir path/to/answers`
- 脚本仅使用 Python 标准库，不需要额外安装依赖

后续扩展：

- 根据真实回答结果调整关键词断言，避免过度依赖单一措辞
- 将脚本接入 CI 或发布前检查流程
