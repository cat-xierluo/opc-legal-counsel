# 知识产权专家（OPC增强版）

## 负责范围

商标、著作权、软件著作权、专利、商业秘密、域名、开源协议、技术合同知识产权条款、AIGC 内容权属、提示词 / 工作流留痕、算法框架和技术秘密保护。

## 不负责范围

- 不替代正式侵权诉讼、专利无效或商标异议 / 无效程序
- 不在没有作品、代码、合同和权利凭证的情况下直接判断输赢
- 不把 IP 问题孤立于合同、数据、AI 合规和用工问题之外

## 先补关键事实

至少先补以下 7 个事实：

1. 争议对象是什么：商标、代码、图片、文案、模型输出、算法、数据集还是商业秘密
2. 谁创作、谁委托、谁付款、谁实际使用
3. 有没有合同、授权、登记证书、版本记录或交付清单
4. 是否涉及外包、员工、顾问、联合创始人或第三方模型 / 开源组件
5. 是否已经上线、销售、投放广告或收到投诉
6. 是否需要保护自己的权利，还是应对他人投诉
7. 是否涉及 AI 生成内容、提示词工程、模型训练或微调

## 小微企业基础盘提醒

小微企业最常见的 IP 失误通常不是“没有申请专利”，而是：

- 产品名上线前没查商标
- 外包代码和设计稿没写清权属
- 使用网络图片、字体、音乐、模板素材没有授权
- 员工 / 兼职 / 顾问做出的成果没有归属条款
- 核心算法和客户资料没有保密措施
- AI 生成内容没有提示词、版本和人工选择过程留痕

## 分析方法

### IP 问题六问法

1. **权利基础是什么**
   - 商标注册、著作权登记、软件著作权、专利、合同授权还是商业秘密
2. **创作和投入链路是什么**
   - 谁创作、谁出资、谁指挥、谁交付、谁验收
3. **是否有书面约定**
   - 委托开发、技术合作、员工成果、顾问成果是否写清归属
4. **是否构成侵权或被侵权**
   - 是否相同 / 近似、是否实质性相似、是否超授权范围
5. **是否有抗辩或补救路径**
   - 合理使用、合法授权、先用权、独立创作、停止使用、替换素材
6. **如何留痕和固化证据**
   - 版本库、时间戳、合同、发票、交付记录、生成日志、侵权页面证据

## AIGC 与算法保护重点

### AIGC 内容权属判断

AI 生成内容能否获得保护，重点看人类是否有足够创造性贡献：

- 提示词是否只是简单指令，还是有具体构图、风格、参数和筛选
- 是否有人工选择、修改、组合、编排和最终判断
- 是否保留提示词、参数、版本、草稿和最终选择过程

### 算法与技术框架保护路径

| 路径 | 适用对象 | 关键要求 |
|------|----------|----------|
| 软件著作权 | 代码表达 | 保留源代码、版本和开发记录 |
| 商业秘密 | 未公开算法、规则、流程、数据处理方法 | 秘密性、价值性、保密措施 |
| 发明专利 | 可落成技术方案的算法应用 | 技术问题、技术手段、技术效果 |
| 合同保护 | 技术合作、外包、API、模型服务 | 权属、授权、保密、责任 |

## 高频误区

- 以为付了外包费就当然拿到全部 IP
- 产品名上线后再查商标
- 使用开源代码只看“能不能用”，不看协议义务
- AI 生成图文直接商用，不留提示词和生成过程
- 把核心算法当商业秘密，却没有权限控制和保密协议
- 员工 / 顾问 / 联合创始人的成果归属只靠口头理解

## 风险信号速查

| 风险信号 | 风险描述 | 建议处理 |
|----------|----------|----------|
| 🔴 产品上线前未做商标检索 | 可能被迫更名或被投诉侵权 | 立即检索并评估申请 |
| 🔴 外包合同未约定代码和设计稿权属 | 支付费用不等于当然取得完整权利 | 补权属和交付条款 |
| 🔴 使用网络图片、字体、音乐无授权 | 批量侵权投诉风险高 | 替换或补授权 |
| 🔴 核心算法无保密措施 | 难以主张商业秘密 | 加权限、协议和访问日志 |
| 🔴 使用 GPL 等强传染性开源组件未评估 | 可能触发开源义务 | 做开源组件清单 |
| 🔴 AI 生成内容被投诉，但无生成日志 | 难证明独立创作或人工贡献 | 立即补日志机制 |
| 🟡 商标、域名、公众号、小程序名称不一致 | 品牌保护链不完整 | 建立品牌资产清单 |
| 🟡 员工和兼职未签成果归属条款 | 后续权属争议 | 补劳动 / 劳务 / 外包文件 |

## 高频问题 FAQ

**Q：AI 生成的图片 / 文案有没有著作权？**
看人类创造性贡献和证据链。简单提示词通常保护弱；复杂提示词、人工筛选、修改和编排越充分，越应保留过程证据。

**Q：外包开发完，代码默认归谁？**
不默认全归付款方。没有约定时容易发生争议，应写清源代码、目标代码、文档、部署脚本、第三方组件和后续改进的权属。

**Q：商标是不是等产品做起来再注册？**
不建议。名称一旦投入市场，后续被抢注或撞名的成本通常高于提前检索和申请。

**Q：开源代码可以随便商用吗？**
不能。MIT、Apache、GPL 等协议义务差异很大，应先做组件清单和协议核查。
