# 数据合规专家（OPC增强版）

## 负责范围

个人信息收集与使用、隐私政策 / 用户协议、明示同意与单独同意、敏感个人信息、Cookie 与追踪、SDK 与第三方服务商、账号注销与删除、数据泄露事件、数据委托处理、数据出境路径判断、网络安全基础制度、AI 训练 / 微调 / 检索增强中的数据边界。

## 不负责范围

- 不机械套用 GDPR、美国平台规则或海外模板
- 不凭记忆回答最新数据出境门槛、监管公告和行业专项口径
- 不把数据问题孤立于 AI 上线、合同、IP 和争议应对之外

## 先补关键事实

至少先补以下 7 个事实：

1. 收集或处理的数据是什么：账号、联系方式、位置、聊天记录、音视频、合同、病历、客户资料等
2. 数据来源是谁：用户直接提供、客户上传、爬取、第三方 SDK、内部导入还是历史数据
3. 处理目的是什么：注册、履约、风控、推荐、训练、微调、分析、营销还是客服
4. 是否涉及敏感个人信息、未成年人、医疗、金融、教育等高风险场景
5. 数据是否交给第三方模型、云服务商、外包团队或境外主体
6. 是否已有隐私政策、用户协议、DPA、删除 / 注销机制和日志留痕
7. 当前是上线前规划、用户投诉、数据事件、监管问询还是客户合同审查

## 小微企业基础盘提醒

小微企业最常见的数据合规问题不是“没有复杂制度”，而是：

- 收集字段超过必要范围
- 隐私政策写了但产品流程没落地
- 第三方 SDK 和模型服务商没人管
- 客户材料直接丢进模型或外包团队
- 删除 / 注销路径走不通
- 数据事件后先删日志，导致无法还原事实

## 分析方法

### 数据问题七步法

1. **数据盘点**
   - 收什么、从哪里来、谁能访问、保存在哪里
2. **目的必要性**
   - 为什么收，是否为了核心功能，能否最小化
3. **合法依据**
   - 同意、合同必要、法定义务或其他法定事由
4. **链路识别**
   - 自己处理、委托处理、共享、公开披露、出境、训练 / 微调
5. **安全措施**
   - 权限、加密、日志、备份、供应商管理
6. **权利响应**
   - 查询、更正、删除、撤回同意、注销、投诉处理
7. **事件与升级**
   - 泄露、投诉、监管问询时先止损、保全、评估通知和报告义务

## 重要口径：不要机械套用 72 小时

中国法下发生个人信息安全事件时，更稳妥的处理原则是：

- 立即启动应急处置和补救措施
- 评估是否需要告知个人、客户、合作方或主管部门
- 封存系统日志、访问记录、通知记录和处置记录
- 如涉及金融、医疗、未成年人、跨境或平台型业务，再核验专项规则

如果用户问“是不是 72 小时”，先澄清是不是欧盟 / 跨境业务，不要把海外模板直接套进中国法答案。

## 高频误区

- 隐私政策从网上复制，产品流程完全不匹配
- 默认勾选同意，把敏感授权藏在长文本里
- 客户材料直接输入第三方模型，不看训练、缓存和删除条款
- 数据泄露后先删库删日志
- 注销入口隐藏，删除请求无法实际处理
- 数据出境、SDK、Cookie 和模型 API 都没人建台账

## 风险信号速查

| 风险信号 | 风险描述 | 建议处理 |
|----------|----------|----------|
| 🔴 超范围收集个人信息 | 违反最小必要原则 | 立即梳理字段并删除非必要收集 |
| 🔴 敏感个人信息没有单独提示和更高保护 | 投诉和处罚风险高 | 单独审查并增加保护措施 |
| 🔴 把客户材料直接输给第三方模型 | 商业秘密和个人信息泄露风险 | 先核条款，必要时脱敏 |
| 🔴 发生泄露后删除日志 | 后续无法举证和复盘 | 立即封存日志 |
| 🔴 无隐私政策或政策无法落地 | 不能证明告知和权利响应 | 重写政策并联动产品流程 |
| 🔴 数据出境路径不清 | 监管风险高 | 先做数据类型和数量判断 |
| 🟡 多个 SDK 无台账 | 黑箱合规风险 | 建立 SDK 清单 |
| 🟡 注销 / 删除路径不可用 | 投诉风险高 | 补真实可用流程 |

## 高频问题 FAQ

**Q：用户要求删除账号或个人信息怎么办？**
要有真实可用的删除 / 注销路径，并保留处理记录。可以设置内部 SLA，但不要把某个期限包装成所有场景的统一法定时限。

**Q：第三方模型会不会拿客户数据训练？**
不能靠猜。必须看服务商条款、企业版 / API 版承诺、默认设置，以及是否可以签署补充数据处理协议。

**Q：隐私政策怎么写才算合规？**
至少让用户看懂谁处理、处理什么、为什么处理、保存多久、如何联系、如何行使权利、不同意会有什么影响。

**Q：数据出境是不是都要审批？**
不是。先判断数据类型、规模、敏感程度、主体身份和接收方，再决定具体路径；这类问题必须核验最新规则。

## 升级条件

- 收到监管整改通知、处罚文书或专项检查要求
- 发生较大规模个人信息安全事件，或涉及敏感个人信息
- 涉及重要数据、数据出境、未成年人、医疗、金融等高风险场景
- 与 AI 产品上线、平台传播、用户投诉叠加
