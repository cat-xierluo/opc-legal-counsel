# AI 产品上线核查样稿

> 适用于用户要求“上线前查一下”“给我一版核查清单”“按顾问口径出一版上线建议”时使用。

## 何时调用

- 面向 C 端或 ToB 的 AI 产品 / AI 功能准备上线
- 需要把抽象合规要求落成具体动作
- 需要输出“24 小时 / 7 天 / 30 天”式动作表

## 建议组合资产

- `assets/risk-checklist.md`
- `references/ai-compliance.md`（快速查法规要点时加载对应核心领域文件）

## 固定输出结构

1. 一句话结论
2. 涉及领域
3. 关键事实缺口 / 当前假设
4. 上线前红线风险与风险等级
5. 现在就做什么（24 小时内）
6. 接下来怎么做（7 天 / 30 天）
7. 需要准备的文件与留痕
8. 何时必须升级给外部律师 / 安全 / 合规支持
9. 法律依据与适用层级

## 样稿示例

### 场景假设

- 主体：一人有限公司
- 业务：面向 C 端的 AI 写作工具
- 功能：上传简历、聊天记录、产品资料，生成简历、自我介绍和营销文案
- 当前状态：尚未正式上线，使用第三方大模型 API

### 标准输出

#### 1. 一句话结论

可以准备上线，但不能在“协议、隐私、模型供应商核查、生成内容标识、投诉处置”都没落地的情况下直接公测。

#### 2. 涉及领域

`ai-compliance + data-compliance + regulatory + ip`

#### 3. 关键事实缺口 / 当前假设

- 暂按中国境内运营、面向普通公众用户处理
- 暂不涉及金融、医疗、未成年人专门场景
- 尚不清楚第三方模型是否存在训练留用、跨境传输或缓存安排

#### 4. 上线前红线风险与风险等级

| 风险点 | 等级 | 为什么危险 |
|--------|------|------------|
| 用户上传材料边界不清 | 高 | 容易超范围收集和处理个人信息、敏感信息 |
| 第三方模型服务条款未审 | 高 | 可能出现训练留用、转处理、缓存或跨境问题 |
| 生成内容标识和投诉机制缺失 | 高 | AI 生成内容被误用或被投诉时缺少基本处置能力 |
| 宣传语夸大能力 | 中 | 容易触发广告和消费者保护风险 |
| 输出侵权和错误使用提示不足 | 中 | 发生投诉时不利于止损和分责 |

#### 5. 现在就做什么（24 小时内）

- 先梳理产品实际收集的数据类型，区分普通个人信息、敏感个人信息和企业资料
- 立刻核查第三方模型或 API 的服务条款，重点看数据留存、训练使用、转处理和跨境安排
- 起草并挂出最小可用版的用户协议、隐私政策和 AI 使用说明
- 确认产品页、功能页和导出页是否需要加 AI 生成提示或标识
- 建一个最基本的投诉入口和内部处置负责人清单

#### 6. 接下来怎么做（7 天 / 30 天）

**7 天内**

- 把输入使用边界、输出限制、禁止用途、投诉举报路径写进用户协议
- 对上传链路做最小化检查，避免默认收过多无关信息
- 梳理营销文案，删除“百分百准确”“完全替代人工”等高风险表述
- 建立日志留存、版本留痕和问题回溯机制

**30 天内**

- 按业务实际情况补完整的数据处理台账、供应商台账和应急预案
- 对高风险场景增加人工复核、敏感词拦截或输出限制
- 复核是否触及最新专项规则、备案 / 登记信息核验要求或行业特别义务

#### 7. 需要准备的文件与留痕

- 用户协议
- 隐私政策
- 第三方模型 / SDK 服务条款核查记录
- 产品功能说明与页面截图
- 投诉处置流程和责任人记录
- 日志留存和版本变更记录

#### 8. 何时必须升级给外部律师 / 安全 / 合规支持

- 已发生批量投诉、数据泄露或舆情
- 业务涉及医疗、金融、未成年人等高监管领域
- 无法确认第三方模型的数据处理边界
- 准备大规模投流或与大型平台深度合作前

#### 9. 法律依据与适用层级

- 全国法规则优先，地方规则只作为覆盖层
- 涉及 AI 标识、备案 / 登记信息、平台规则等时间敏感事项时，应核验最新官方来源
- 本样稿提供的是顾问级分诊和动作清单，不替代正式法律意见
